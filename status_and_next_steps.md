# 1on1 Cloud System - 開発の現状と次回のアクションプラン

本ドキュメントは、ここまでの開発内容のまとめと、次回「クラウド化（データベース導入）」へ進むための引き継ぎ資料です。次回のセッション開始時に、AIにこの情報を参照させることでスムーズに続きから再開できます。

## 📁 開発状況（フロントエンド完成）

現在のシステムは、見た目（UI）と動作（UX）を**完全に構築し終えたプロトタイプ版（v5）**です。
*   **デザイン**: アブストラクトなメッシュグラデーション＋ガラス風UI（Glassmorphism）の「プレミアムなログイン画面」を採用。
*   **実装スタック**: ピュアな `HTML` + `CSS` (Vanilla) + `JavaScript`。
*   **データ保存形式**: 現在はブラウザの『ローカルストレージ (`1on1_appState_v4`)』内に疑似的なデータベースとして保存しています。

### 実装済みの主要機能
1.  **ログイン・認証システムとパスワード制限**（テスト用: IDを選択し、パスワードは全員「1234」で通過するよう仮設定）
2.  **管理画面（要パスワード: 123）の構築**
    *   社員情報の追加／削除、および「システム管理者権限」の個別付与。
    *   1on1ペア（誰が誰の上司か）の紐付け設定。
    *   メンバーの事前提出用アンケート（質問）の動的追加／削除。
3.  **役割に応じたダッシュボード・過去履歴の動的出し分け（アクセス制御）**
    *   ログインしたユーザーの権限に応じて「自分が受ける1on1」「部下に行う1on1」の表示を完全分離。
    *   他人の面談履歴・プライベートメモは閲覧不可（※マスター管理者除く）。

## 🚀 次回のアクションプラン：完全なクラウド化へ

次回は、いよいよ現在の `app.js` の localStorage 保存部分を取り除き、**本当のクラウド・データベース（Supabase）**へ直結させます。

### 直近で行う3つのタスク

#### 1. Supabaseプロジェクトの立ち上げ（ユーザー様作業）
ユーザー様ご自身で [Supabase](https://supabase.com/) アカウントを作成し、新規プロジェクト（例: `1on1-cloud`）を構築してください。
取得された以下の情報をAIに教えていただきます。
*   `Project URL`
*   `anon public key`

#### 2. データベースのテーブル設計と構築（AI作業）
AIがSupabase側に必要な箱（テーブル）をSQLで自動生成します。
*   `users` テーブル（社員ID、名前、管理者フラグ）
*   `pairings` テーブル（上長ID、メンバーID）
*   `questions` テーブル
*   `sessions` / `history` テーブル（面談内容、メモ、状況）

#### 3. プログラム (app.js) のクラウド通信化・デプロイ（AI連携作業）
*   HTMLにSupabase JSライブラリを組み込み、app.jsを「ローカルストレージ版」から「ネットワーク通信版（非同期 fetch/await）」に書き換えます。
*   完成後、フロントエンドを Vercel もしくは Cloudflare Pages にデプロイし、**全世界からアクセス可能＆共有可能なシステムのURL** を発行・公開します！

**次回のAIへのメッセージ例:**
